Functional encryption: All user data will be stored and calculated in Ruby's detailed functional encryption algorithm

Data Marketplace: Ruby Protocol must propose a solution on how all entities will participate in data transactions and transaction execution process. Ruby Protocol proposes a new watermarking technique and functional coding method to build a big data market by leveraging the power of blockchain technology and smart contracts, on top of existing watermarking techniques, in detail level. The proposed system enables critical operations in big data scenarios, including data collection, aggregation, storage, sale and purchase.Ruby Protocol establishes an access control mechanism to ensure ensure that legitimate buyers, upon receiving the permission of the data seller, can access the destination data Ruby Protocol introduces a payment system that allows data transactions between sellers and buyers in the marketplace big data field. A review mechanism, with buyer incentive, is also introduced to attract more sellers and buyers in the system, and motivate them to behave honestly.Ruby Protocol details the approach of Ruby Protocol overcomes current challenges in big data markup implementations and enables a transparent, immutable audit trail to the movement of data in the big data marketplace. Finally, Ruby Protocol proposes a detailed access control policy to be integrated into the Substrate module and data monetization will also be made through the Substrate module.
